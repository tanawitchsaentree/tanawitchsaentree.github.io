## Overview

A commercial claims platform at a leading European insurer — multi-line, high-complexity, six-figure reserves per case. Marine. Property. Liability. Cyber. Engineering. The kind of system where a misaligned button state isn't UX debt — it's a legal exposure.

The job was design systems lead. The job changed shape. I stopped shipping Figma and started shipping verified, working software straight from the ticket. Nobody asked me to. I decided the output needed to change.

## What I Built

- FNOL wizard, sections and coverage review, closure automation, claim reopen, notes, financial overview, restrictions, activities log, payment and provider modals — 15+ features in four months, each ticket-to-live in one session
- Established an audit → build → verify loop where no single pass does more than one thing: audit reads, build writes, verify runs the app and screenshots the evidence
- Extracted every failure pattern into a skill library loaded before each new session — persistent memory that prevents repeating the same six debugging rounds

## The Outcome

Features shipped the same day tickets arrived. The prototype became the backend API contract — the backend team designed their schemas from what the running interface showed. The design artifact was the running software.

The title didn't change. The output did.
